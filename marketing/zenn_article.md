### なぜStripeじゃなくてPAY.JPか

Stripeは海外企業のため、日本の個人事業主だと審査が厳しいケースがあります。PAY.JPは国内決済で、審査が比較的通りやすいです。

APIの使い方も違います：

```typescript
// PAY.JP: Basic認証
const auth = Buffer.from(`${process.env.PAYJP_SECRET_KEY}:`).toString("base64");
const res = await fetch("https://api.pay.jp/v1/subscriptions", {
  method: "POST",
  headers: {
    "Authorization": `Basic ${auth}`,
    "Content-Type": "application/x-www-form-urlencoded",
  },
  body: new URLSearchParams({ customer: customerId, plan: planId }),
});
```

### Claude APIのコスト最適化

15本全部でHaikuを使っています。GPT-4oやClaude Sonnetを使うとAPIコストが爆発するので。

```typescript
const response = await anthropic.messages.create({
  model: "claude-haiku-4-5-20251001", // 安い・速い・十分な品質
  max_tokens: 2000,
  messages: [{ role: "user", content: prompt }],
});
```

Haikuで1リクエスト約¥0.5〜2程度。ユーザーが月100回使っても¥200のAPIコスト。

### タブUI（全サービス共通パターン）

複数の出力を1画面で見せるためのUIパターン。コンテンツを`===TAG===`で区切って、フロントでパース：

```typescript
// APIレスポンス例
"=== LEGAL ===\n法的評価の内容...\n=== EVIDENCE ===\n証拠収集GL..."

// パース
function parseResult(text: string) {
  const tags = ["LEGAL", "EVIDENCE", "CERTIFIED", "REPORT"];
  return tags.reduce((acc, tag) => {
    const re = new RegExp(`===\\s*${tag}\\s*===([\\s\\S]*?)(?===|$)`, "i");
    const m = text.match(re);
    acc[tag] = m ? m[1].trim() : "";
    return acc;
  }, {} as Record<string, string>);
}
```

## やらかした失敗3選

### 失敗1: 景品表示法を知らずに虚偽数値を書いていた

「2,400+事業者が利用中」「98%が対応時間半減」などのコピーをLPに書いていました。

ユーザーが1人もいないのに。

景品表示法の**不当表示（優良誤認）**に該当します。実際に行政指導を受けた事例もあります。全部削除しました。

**教訓: 「それっぽいLP」を作る前に景品表示法を勉強する。**

### 失敗2: 弁護士法72条を知らなかった

パワハラ対策AIのヒーローコピーが「あなたの状況は、法的にアウトです。」でした。

ユーザーの状況を聞く前から法的判定を下す、というUIになっていました。

弁護士法72条は「弁護士でない者が法律事務を取り扱うこと」を禁じています。AIが法的判断を下すような表現は非弁行為のリスクがあります。

全コピーを書き直して、目立つ位置に免責バナーを設置しました。

**教訓: 法律・医療・金融系サービスは作る前に弁護士法・薬機法・金商法を調べる。**

### 失敗3: 「作ること」に集中しすぎた

15本デプロイするのに約3ヶ月かかりました。

最初の1本を作った後、コールドDMを5社に送るだけでよかったんです。

でも「もっと良いサービスを」「あのジャンルも」と次々作り続けて、営業をほぼしませんでした。

**教訓: 1本作ったら、2本目を作る前に10社に営業する。**
